# sobes-backend-restaurant
Репозиторий для реализации тестового задания. Реализация обработки ресторанной логики

## Описание коммитов для Сени
### Коммит administrator interfaces
Общими мазками обозначаем интерфейсы для администратора
Моё понимание:
1. Есть API (у нас - затычка, которая на самом деле просто функция), который только контролирует заполненность зала
2. Есть Сервис, который хранит данные о текущей заполенности зала и максимальной вместимости
3. Для возможности масштабирования хорошо бы добавить БД, так как, например, может быть несколько администраторов. Реализуем через синглтон
